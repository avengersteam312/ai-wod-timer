# Promptfoo Eval Config Author

You are writing or editing a promptfoo YAML test config in `promptfoo/{type}.yaml`.

## File Structure

```yaml
description: 'TYPE Workout Parser Tests'

prompts:
  - file://prompts/{type}.txt   # generated by scripts/generate_prompts.py

providers:
  - id: openai:gpt-4o
    config:
      max_tokens: 2000
      response_format:
        type: json_object          # forces JSON output

tests:
  - description: "Human-readable test name"
    vars:
      workout_text: |
        AMRAP 20min:
        10 Wall Balls
        10 Box Jumps
    assert:
      - type: is-json              # ALWAYS first
      - type: javascript
        value: 'JSON.parse(output).workout_type === "amrap"'
      - type: javascript
        value: 'JSON.parse(output).intervals.length === 1'
      - type: javascript
        value: 'JSON.parse(output).intervals[0].duration === 1200'
      - type: javascript
        value: 'JSON.parse(output).movements.length === 3'
```

## Assertion Rules

1. **Always start with `is-json`** — if output isn't JSON, all downstream assertions fail gracefully
2. **Check `workout_type` second** — confirms classifier routed correctly
3. **Check `intervals.length`** — most common failure point
4. **Check specific `duration` values** — not just existence
5. **Check `movements` content** with `.some(m => m.weight)` or `.some(m => m.distance)`

## Per-Type Interval Count Logic

| Type | Count formula |
|------|--------------|
| AMRAP (single) | 1 |
| AMRAP (with rest) | work + rest intervals, no trailing rest |
| EMOM N min | N intervals |
| E2MOM N min | N/2 intervals |
| E3MOM N min | N/3 intervals |
| FOR_TIME | 1 (+ rest intervals if sets with rest stated) |
| TABATA (single, 8 rounds) | 16 (8 work + 8 rest) |
| INTERVALS | 2 × rounds (alternating work/rest) |
| STOPWATCH | 1 with duration=0 |

## Standard Test Case Coverage Per Type

Write at least these test scenarios:

1. **Basic** — most common format, explicit duration
2. **Weighted** — movements with weight (test `m.weight` assertion)
3. **Distance/Calories** — test `m.distance` or `m.calories` assertion
4. **No explicit duration** — for FOR_TIME without cap, duration should be 0
5. **Named complex** — e.g., "Cindy" or "DT" — verify movements are extracted, not complex name
6. **Edge case** — for EMOM: repeat:true; for TABATA: custom rounds; for AMRAP: multiple with rest

## JavaScript Assertion Examples

```yaml
# workout_type check
value: 'JSON.parse(output).workout_type === "emom"'

# interval count
value: 'JSON.parse(output).intervals.length === 10'

# all intervals have same duration (EMOM)
value: 'JSON.parse(output).intervals.every(i => i.duration === 60)'

# filter by type
value: 'JSON.parse(output).intervals.filter(i => i.type === "work").length === 3'

# movement has weight
value: 'JSON.parse(output).movements.some(m => m.weight)'

# movement has distance
value: 'JSON.parse(output).movements.some(m => m.distance)'

# movement has calories
value: 'JSON.parse(output).movements.some(m => m.calories === 12)'

# movement duration (time-based movement, not interval duration)
value: 'JSON.parse(output).movements.some(m => m.duration === 30)'

# repeat flag (EMOM ladder)
value: 'JSON.parse(output).intervals[0].repeat === true'

# stopwatch (duration = 0)
value: 'JSON.parse(output).intervals[0].duration === 0'
```

## Common Mistakes

- **Off-by-one interval count**: EMOM 10min = 10 intervals (not 9 or 11)
- **Missing `is-json` guard**: if LLM returns non-JSON, JS assertions throw instead of failing cleanly
- **Duration in minutes not seconds**: 20min = 1200, not 20
- **Tabata round count**: 8 rounds = 16 interval objects (not 8)
- **FOR_TIME with cap**: `duration === cap_in_seconds` (not 0)
- **FOR_TIME without cap**: `duration === 0` (stopwatch mode)
