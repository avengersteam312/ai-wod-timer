## Codebase Patterns
- Frontend lives in `/frontend` directory, not at project root
- Config files live in `src/config/` (e.g., supabase.ts) - Firebase has been removed
- Environment variables use VITE_ prefix for Vite to expose them
- Pre-existing TypeScript errors exist in the codebase (unrelated to new changes) - ignore when checking new file typechecks
- Supabase migrations go in `/supabase/migrations/` with format `YYYYMMDDHHMMSS_description.sql`
- Use `handle_updated_at()` trigger function for auto-updating timestamps (created in profiles migration)
- Pinia stores live in `src/stores/` and use composition API style (setup function with refs/computed)
- Supabase auth types come from `@supabase/supabase-js`: User, Session, AuthError, Subscription
- Run typecheck with `npx vue-tsc --noEmit` (no npm script for typecheck)
- Supabase auth store uses `loading` property (not `isLoading`)
- For API auth, get token from `authStore.session.access_token`
- Supabase auth methods return `{ success, error }` objects - use result-based error handling
